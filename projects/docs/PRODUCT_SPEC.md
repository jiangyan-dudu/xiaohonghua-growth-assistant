# 《每天一朵小红花》产品需求与设计文档

## 一、产品概述

### 1.1 产品定位
《每天一朵小红花》是一款面向小学生的好习惯养成激励APP。通过游戏化的积分体系、奖励商城和抽奖功能，帮助孩子建立良好的日常习惯，同时让家长能够轻松管理和引导孩子的行为。

### 1.2 目标用户
- **主要用户**：小学生（6-12岁）
- **管理用户**：家长

### 1.3 核心理念
- **正向激励**：通过积分奖励鼓励孩子完成任务
- **亲子互动**：家长审核、确认任务，共同参与成长
- **游戏化体验**：抽奖、等级系统增加趣味性

### 2.4 奖励模块

#### 2.4.1 顶部Tab（固定）
- **奖励商城**：积分兑换奖励
- **幸运抽奖**：消耗积分抽奖

#### 2.4.2 奖励等级与积分规则
| 等级 | 积分范围 | 示例奖励 |
|------|----------|----------|
| 小奖励(low) | 1-50积分 | 玩游戏10分钟、看电视10分钟 |
| 中奖励(medium) | 51-200积分 | 零食10元、下棋20分钟 |
| 大奖励(high) | 201+积分 | 电影观影、甜品、出游 |

#### 2.4.3 兑换流程
1. 用户选择奖励
2. 确认兑换（扣除积分）
3. 生成兑换记录
4. 家长兑现奖励

#### 2.4.4 抽奖规则
| 参数 | 说明 |
|------|------|
| 抽奖消耗 | 50积分/次 |
| 奖品等级 | 普通、稀有、传说 |

#### 2.4.5 抽奖概率分配
| 等级 | 总概率 | 分配规则 |
|------|--------|----------|
| 普通(common) | 80% | 同等级奖品均分 |
| 稀有(medium) | 17% | 同等级奖品均分 |
| 传说(rare) | 3% | 同等级奖品均分 |

**概率自动计算说明**：
- 新增/修改/删除奖品后，系统自动重新计算所有奖品概率
- 同等级奖品均分该等级的总概率
- 示例：3个普通奖品，每个概率 = 80% ÷ 3 ≈ 26.67%

#### 黏土阴影（核心特征）
```css
/* 卡片阴影 */
shadowColor: '#7C5CFC';
shadowOffset: { width: 0, height: 8 };
shadowOpacity: 0.15;
shadowRadius: 20;
elevation: 6;

/* 按钮阴影 */
shadowColor: '#5A3ED9';
shadowOffset: { width: 0, height: 6 };
shadowOpacity: 0.3;
shadowRadius: 12;
elevation: 6;
```

#### 有色阴影原则
- **禁止**使用纯黑阴影
- **必须**使用与主色调一致的颜色作为阴影色
- 阴影透明度：0.06 ~ 0.15

## 七、交互规范

### 7.1 列表分页
- 默认显示5条记录
- 点击"查看更多"每次加载5条

### 7.2 Tab固定规则
- 底部Tab导航：始终固定
- 页面内Tab：固定在页面顶部，不随内容滚动
- Tab切换后：滚动条回到顶部

### 7.3 弹窗交互
- 使用系统原生Alert（移动端）
- 使用window.confirm（Web端）

### 7.4 表单验证
- 奖励积分与等级自动关联
- 抽奖概率自动计算
- 每日申请任务限制（10个）

## 八、安全与权限

### 8.1 家长密码
- 默认密码：`000000`
- 可在家长后台修改

### 8.2 数据隔离
- 所有数据存储在本地AsyncStorage
- 无网络传输，保护隐私
